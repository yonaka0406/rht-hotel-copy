# Glossary

This document provides definitions for technical terms and abbreviations used throughout the VPS configuration documentation.

## A

**Apache** - Open-source web server software used to serve web pages and handle HTTP requests.

**API** - Application Programming Interface; a set of protocols and tools for building software applications.

**Autovacuum** - PostgreSQL's automatic maintenance process that cleans up dead tuples and updates table statistics.

## B

**Backup** - A copy of data stored separately from the original to protect against data loss.

**Bash** - Bourne Again Shell; a command-line interface and scripting language for Unix-like operating systems.

**Bot** - Automated software program that performs repetitive tasks, often used for web scraping or attacks.

## C

**CLI** - Command Line Interface; a text-based interface for interacting with computer systems.

**Connection Pooling** - A technique to manage database connections efficiently by reusing existing connections.

**CPU** - Central Processing Unit; the main processor of a computer that executes instructions.

**CRUD** - Create, Read, Update, Delete; the four basic operations for persistent storage.

## D

**Database** - An organized collection of structured information or data stored electronically.

**DoS** - Denial of Service; an attack that makes a service unavailable to legitimate users.

**DDoS** - Distributed Denial of Service; a DoS attack using multiple compromised systems.

**DNS** - Domain Name System; translates domain names to IP addresses.

## E

**Encryption** - The process of converting data into a coded format to prevent unauthorized access.

**Environment Variable** - A dynamic value that affects running processes on a computer system.

## F

**Fail2ban** - Intrusion prevention software that protects against brute-force attacks by blocking IP addresses.

**Firewall** - A network security system that monitors and controls incoming and outgoing network traffic.

**FTP** - File Transfer Protocol; a standard network protocol for transferring files.

## G

**GB** - Gigabyte; a unit of digital information equal to 1,024 megabytes.

**Git** - A distributed version control system for tracking changes in source code.

**GUI** - Graphical User Interface; a visual interface that allows users to interact with software.

## H

**HTTP** - Hypertext Transfer Protocol; the foundation of data communication on the web.

**HTTPS** - HTTP Secure; HTTP with encryption using SSL/TLS protocols.

**Health Check** - A monitoring process that verifies the operational status of a system or service.

## I

**IP Address** - Internet Protocol address; a unique identifier for devices on a network.

**iptables** - A firewall utility for Linux systems that controls network traffic.

**I/O** - Input/Output; the communication between a computer system and the outside world.

## J

**JSON** - JavaScript Object Notation; a lightweight data interchange format.

**Journal** - In systemd, a centralized logging system that collects and stores log data.

## K

**Kernel** - The core component of an operating system that manages system resources.

## L

**Load Average** - A measure of system activity representing the average number of processes using the CPU.

**Log** - A record of events that occur in a computer system or application.

**Log Rotation** - The process of archiving and managing log files to prevent disk space exhaustion.

## M

**MB** - Megabyte; a unit of digital information equal to 1,024 kilobytes.

**Memory** - Computer storage used to hold data and programs currently in use.

**Monitoring** - The process of observing and checking system performance and health.

## N

**Network** - A collection of interconnected devices that can communicate with each other.

**Nginx** - A web server and reverse proxy server known for high performance and low resource usage.

## O

**OS** - Operating System; software that manages computer hardware and software resources.

**OOM** - Out of Memory; a condition where a system runs out of available memory.

## P

**PgBouncer** - A connection pooler for PostgreSQL databases.

**PID** - Process Identifier; a unique number assigned to each running process.

**PostgreSQL** - An open-source relational database management system.

**Process** - A running instance of a computer program.

## Q

**Query** - A request for information from a database.

**Queue** - A data structure that follows First-In-First-Out (FIFO) principle.

## R

**RAM** - Random Access Memory; volatile computer memory used for temporary data storage.

**Recovery** - The process of restoring a system to a functional state after a failure.

**Replication** - The process of copying data from one database to another for backup or distribution.

**Root** - The superuser account in Unix-like systems with full administrative privileges.

## S

**Sakura VPS** - Virtual Private Server service provided by Sakura Internet Inc.

**SCP** - Secure Copy Protocol; a method for securely transferring files over a network.

**Service** - A background process that runs continuously to provide specific functionality.

**SSH** - Secure Shell; a cryptographic network protocol for secure remote access.

**SSL** - Secure Sockets Layer; a security protocol for establishing encrypted connections.

**SSD** - Solid State Drive; a storage device with no moving parts, faster than traditional hard drives.

**Systemd** - A system and service manager for Linux operating systems.

## T

**TCP** - Transmission Control Protocol; a reliable, connection-oriented network protocol.

**TLS** - Transport Layer Security; the successor to SSL for secure communications.

**TTL** - Time To Live; the maximum time data can exist before being discarded.

## U

**Ubuntu** - A Linux distribution based on Debian, known for ease of use.

**UFW** - Uncomplicated Firewall; a user-friendly interface for managing iptables.

**URL** - Uniform Resource Locator; the address of a resource on the internet.

**User Agent** - A string that identifies the client software making HTTP requests.

## V

**Vacuum** - A PostgreSQL maintenance operation that reclaims storage and updates statistics.

**VNC** - Virtual Network Computing; a system for remote desktop access.

**VPS** - Virtual Private Server; a virtualized server environment.

## W

**WAL** - Write-Ahead Logging; PostgreSQL's method for ensuring data integrity.

**Web Server** - Software that serves web content to clients over HTTP/HTTPS.

## X

**XML** - eXtensible Markup Language; a markup language for storing and transporting data.

## Y

**YAML** - YAML Ain't Markup Language; a human-readable data serialization standard.

## Z

**Zone** - In the context of Sakura VPS, a geographical location where servers are hosted.

## Technical Concepts

### Database Terms

**ACID** - Atomicity, Consistency, Isolation, Durability; properties that guarantee database transactions are processed reliably.

**Index** - A database structure that improves query performance by providing fast access paths to data.

**Schema** - The structure or blueprint of a database, defining tables, fields, and relationships.

**Transaction** - A sequence of database operations that are treated as a single unit of work.

**Tuple** - A single row or record in a database table.

### Network Terms

**Bandwidth** - The maximum rate of data transfer across a network connection.

**Latency** - The time delay between sending a request and receiving a response.

**Port** - A communication endpoint in networking, identified by a number.

**Protocol** - A set of rules governing data communication between devices.

**Subnet** - A subdivision of an IP network.

### Security Terms

**Authentication** - The process of verifying the identity of a user or system.

**Authorization** - The process of determining what actions an authenticated user can perform.

**Brute Force Attack** - An attack method that tries many passwords or keys until the correct one is found.

**Encryption** - The process of converting data into a coded format to prevent unauthorized access.

**Hash** - A fixed-size string generated from input data, used for data integrity verification.

**Intrusion Detection** - The process of monitoring network traffic for suspicious activities.

**Vulnerability** - A weakness in a system that can be exploited by attackers.

### System Administration Terms

**Cron** - A time-based job scheduler in Unix-like operating systems.

**Daemon** - A background process that runs continuously without user interaction.

**Environment** - The context in which a program runs, including variables and settings.

**Log Level** - A classification system for log messages based on severity (DEBUG, INFO, WARN, ERROR, FATAL).

**Mount** - The process of making a file system accessible at a specific directory.

**Package Manager** - Software that automates the installation, upgrade, and removal of software packages.

**Repository** - A storage location for software packages or source code.

**Service** - A program that runs in the background to provide specific functionality.

**Shell** - A command-line interface that allows users to interact with the operating system.

**Symbolic Link** - A file that points to another file or directory.
